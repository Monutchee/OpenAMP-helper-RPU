# libopenamp-rpu

Generic OpenAMP remote-side (RPU) helper library for AMD/Xilinx SoCs.
Wraps the AMD OpenAMP/remoteproc platform glue in a small, exception-free
C++ API (`mnc::OpenAmpPlatform`, `mnc::RpmsgEndpoint`) safe for
bare-metal / FreeRTOS use. Carries **no board-specific addresses** — every
machine supplies its channel configuration as a generated header (see
*Port contract* below).

```
include/mnc/          C++ API (openamp_platform.hpp, rpmsg_endpoint.hpp)
src/                  C++ implementation
machine/zynqmp_r5/    AMD platform glue for ZynqMP Cortex-R5 (BSD-3-Clause,
                      derived from OpenAMP/openamp-system-reference).
                      Future SoC ports (versal_r5, ...) live as siblings.
```

## Port contract (per machine, per core)

The library is configured entirely by `amd_platform_info.h`, generated by
the **lopper openamp assist** from the same domain YAML that produces the
Linux device tree — firmware and kernel therefore cannot drift. Generation
(one run per remote core):

```sh
export LOPPER_DTC_FLAGS="-b 0 -@"
lopper -f --enhanced --permissive -O <outdir> \
  <domain-or-system dts with /domains openamp relations> -- openamp \
  --openamp_header_only \
  --openamp_output_filename=<outdir>/amd_platform_info.h \
  --openamp_remote=<esw machine, e.g. psu_cortexr5_0>
```

The consuming build must:

1. compile all library sources (src/*.cpp + machine/<soc>/*.c — the C files
   must be compiled as C, not C++),
2. add include paths: `include/`, `machine/<soc>/`, and the directory
   containing the generated `amd_platform_info.h` for **this core**,
3. define `-D_AMD_GENERATED_` (and build against a FreeRTOS BSP that
   defines `FREERTOS_BSP`/`USE_FREERTOS`, so the generated IRQ vector is
   the RTOS-adjusted one).

Required macros (all provided by the generated header; a port may instead
pass them as `-D` flags): `SHARED_MEM_PA`, `SHARED_MEM_SIZE`,
`SHARED_BUF_OFFSET`, `IPI_CHN_BITMASK`, plus `IPI_IRQ_VECT_ID` /
`POLL_BASE_ADDR` (default to the BSP's `XPAR_XIPIPSU_0_*` when absent).
A missing channel config **fails the build** (`#error`) — there are no
silent defaults.

Known-good generator: lopper v1.3.0 (xlnx v2026.1) **with** the two fixes
carried in `meta-monutchee/meta-zynqmp-addon/recipes-kernel/lopper/`
(dual-R5 split mode; header-only generation for the 2nd core and
`SHARED_MEM_SIZE` semantics). Stock lopper silently generates nothing for
the second core and emits a wrong `SHARED_MEM_SIZE`.

## Consumers

- Vitis (Empty-app cmake): list the sources and include dirs in
  `UserConfig.cmake` — see `KR260Demo_RPU` for a working dual-core example.
- Yocto: fetch with `gitsm://` if this repo is consumed as a submodule.
